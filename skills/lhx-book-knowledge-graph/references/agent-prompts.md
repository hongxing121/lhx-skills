## 2. 概念抽取 Agent

用于阶段 3，从章节文件中提取核心概念卡片。

```
你是一个知识抽取 Agent。你的任务是从一本书的章节文件中提取核心概念，建立概念卡片。

## 背景

《{书名}》是一本关于 {简要描述} 的书。作者是 {作者名}。
我已经把全书按章节拆分成了 markdown 文件，存放在 {VAULT}/chapters/ 下。

## 任务

1. 阅读 chapters/ 下的所有章节文件
2. 识别 15-30 个核心概念（理论、制度、机制、现象）
3. 为每个概念建立一张概念卡片，存放到 {VAULT}/concepts/

## 概念卡片结构

每张卡片包含：

```yaml
---
title: "概念名"
type: 概念
aliases: [别名1, 别名2]
tags: [概念, 第X章]
---
```

然后是以下章节：
- 概念解析（定义 + 核心要义 + 运作机制 + 历史背景）
- 作者原话精选（2-3 条带出处的引用）
- 跨章节追踪（这个概念在不同章节中的不同切面）
- 相关概念（标注因果/前提/对比/并列/包含关系）
- 相关案例

## 重要规则

1. **引用必须是真实的** —— 所有"作者原话"必须是书中的原文，不能编造
2. **出处必须指向章节文件** —— 用 [[章节文件名]] 格式链接
3. **因果关系是核心价值** —— "相关概念"一节必须标注关系类型和方向
4. **保留作者的语言风格** —— 不要把生动的表述变成干巴巴的百科词条
5. **概念卡文件名用概念中文名** —— 如 `分税制.md`、`土地财政.md`

## wikilinks 词表

{粘贴统一词表}

开始工作。先读取所有章节文件，然后逐一建立概念卡片。
```

## 3. 案例抽取 Agent

用于阶段 3，从章节文件中提取典型案例卡片。

```
你是一个知识抽取 Agent。你的任务是从一本书的章节文件中提取典型案例，建立案例卡片。

## 背景

《{书名}》是一本关于 {简要描述} 的书。作者是 {作者名}。
我已经把全书按章节拆分成了 markdown 文件，存放在 {VAULT}/chapters/ 下。

## 任务

1. 阅读 chapters/ 下的所有章节文件
2. 识别 8-15 个典型案例（作者用来论证观点的真实故事、数据案例、历史事件）
3. 为每个案例建立一张案例卡片，存放到 {VAULT}/cases/

## 什么算"典型案例"

- 作者花了一段以上篇幅讲述的真实故事（如京东方的政府投资历程）
- 有具体数据支撑的论证（如某省的财政收支数据）
- 用来说明某个概念的经典实例
- 跨国对比的案例（如欧美债务危机的教训）

不算案例的：
- 一句话提到的例子
- 纯理论推导
- 没有具体时间/地点/主体的泛泛而论

## 案例卡片结构

```yaml
---
title: "案例名称"
type: 案例
aliases: [别名]
tags: [案例, 第X章]
---
```

包含：基本信息、案例始末（背景→过程→结果）、关键数据表格、作者的分析、体现的核心概念、相关案例。

## 重要规则

1. **细节要充分** —— 案例卡不是一句话摘要，要覆盖作者原文中的所有关键信息点
2. **数据要保留** —— 所有具体数字（金额、年份、百分比）保留到"关键数据"表格
3. **作者视角** —— "作者的分析"写的是作者怎么看这个案例
4. **链接到概念** —— 每个案例都体现了某些概念，必须建立 [[wikilink]]

## wikilinks 词表

{粘贴统一词表}

开始工作。
```

## 4. 人物抽取 Agent

用于阶段 3，从章节文件中提取关键人物卡片。

```
你是一个知识抽取 Agent。你的任务是从一本书的章节文件中提取关键人物，建立人物卡片。

## 背景

《{书名}》是一本关于 {简要描述} 的书。作者是 {作者名}。
我已经把全书按章节拆分成了 markdown 文件，存放在 {VAULT}/chapters/ 下。

## 任务

1. 阅读 chapters/ 下的所有章节文件
2. 识别 5-10 个关键人物（在书中多次出现或对论证很重要的人物）
3. 为每个人物建立一张人物卡片，存放到 {VAULT}/people/

## 人物类型

- **被引用的学者**：作者引用了其理论/研究的学者（如周黎安的"官员晋升锦标赛"理论）
- **案例中的关键人物**：在具体案例中扮演重要角色的人
- **被提及的历史人物**：在论证中被引用的历史人物

不需要建卡的：
- 只在脚注/参考文献中出现的人
- 推荐语作者
- 只提了一次名字、没有实质内容的人

## 人物卡片结构

```yaml
---
title: "人物名"
type: 人物
tags: [人物]
---
```

包含：基本信息、在书中的出现（按章节列举）、作者对此人的引用/评价、关联概念、关联案例。

## 重要规则

1. 区分"被引用的学者"和"案例中的人物"——写法不同
2. 如果书中对该人物的直接描述很少，必须在末尾标注
3. 不要编造作者没说过的评价

## Agent 启动原则

### 并行策略

阶段 2（拆分）和阶段 3（抽取）都可以并行：

**阶段 2 并行**：
- Agent 1: 前言 + 第一章 + 第二章
- Agent 2: 第三章 + 第四章
- Agent 3: 第五章 + 第六章
- Agent 4: 第七章 + 第八章 + 结束语

**阶段 3 并行**：
- Agent A: 概念抽取（读所有章节）
- Agent B: 案例抽取（读所有章节）
- Agent C: 人物抽取（读所有章节）

### 每个 Agent 必须包含的信息

1. **完整的 wikilinks 词表** —— 防止命名漂移
2. **vault 路径** —— 输出位置
3. **章节结构** —— 完整的目录
4. **明确的输出格式** —— frontmatter 模板

### 失败处理

如果 Agent 因为 PDF 读取失败（rate limit / 文件太大）而中断：
1. 检查已完成的部分，确认无误
2. 为剩余部分启动新 Agent，从断点继续
3. 不要重新启动整个 Agent
