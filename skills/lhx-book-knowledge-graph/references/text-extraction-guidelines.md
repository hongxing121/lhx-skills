# 文本提取与章节拆分指南

这个阶段是书籍知识图谱独有的，替代名人版的"素材收集 + 翻译"。目标：把一本书的 PDF 变成结构化的 markdown 章节文件。

---

## PDF 文本提取

### 方法选择

| PDF 类型 | 判断方法 | 提取工具 |
|---------|--------|---------|
| 文字版 PDF | Read 工具能直接读出文字 | Read 工具（每次最多 20 页） |
| 扫描版/影印版 PDF | Read 工具只能看到图片 | `pdf-ocr-skill` 或 `qianfanocr-document-intelligence` |
| epub / mobi | - | 先用 calibre 命令行转 PDF，或直接读取 |

### Read 工具分批策略

PDF 每次最多读 20 页。对于 300 页的书，需要分 15 批次：

```
Read pages 1-20    → 扉页、目录、前言
Read pages 21-40   → 前言续 + 第一章开头
Read pages 41-60   → 第一章续
...
```

**关键技巧**：第一批先读目录页，确认每章的起始页码。然后按章节边界来分批，避免在章节中间截断。

### 并行读取

可以启动多个 Agent 并行读取不同页码范围，但每个 Agent 必须拿到完整的章节结构和 wikilinks 词表。

## 章节拆分规范

### 文件命名

```
<章序号>-<节序号> <章标题> - <节标题>.md
```

示例（《置身事内》）：
```
00 前言 从了解现状开始.md
01-01 地方政府的权力与事务 - 政府治理的特点.md
01-02 地方政府的权力与事务 - 外部性与规模经济.md
01-03 地方政府的权力与事务 - 复杂信息.md
01-04 地方政府的权力与事务 - 激励相容.md
01-05 地方政府的权力与事务 - 招商引资.md
01-06 地方政府的权力与事务 - 结语.md
02-01 财税与政府行为 - 分税制改革.md
02-02 财税与政府行为 - 土地财政.md
...
```

**序号规范**：
- 前言/结束语等特殊章节用 `00`、`09` 等
- 章用两位数：`01`-`08`
- 节用两位数：`01`-`05`
- 结语算作该章最后一节

### frontmatter 格式

```yaml
---
title: "财税与政府行为 - 分税制改革"
type: 章节
chapter: 2
section: 1
tags: [章节, 第二章, 上篇]
---
```

### 内容处理规则

1. **保留原文** —— 章节文件的主体内容就是作者原文。不要改写、删减或"提炼"
2. **wikilinks 标注** —— 遇到关键概念、人物、案例时，用 `[[双括号]]` 标注
3. **脚注处理** —— 书中的脚注/尾注保留在正文中，用括号或引用格式标注
4. **图表处理** —— 如果书中有图表（如组织架构图、数据表格），用文字描述或 markdown 表格还原
5. **扩展阅读** —— 每章末尾的"扩展阅读"推荐也保留，但不需要 wikilink 标注

### wikilinks 标注规范

在拆分过程中，识别并标注以下内容：

- **核心概念**：`[[分税制]]`、`[[土地财政]]`、`[[城投公司]]`
- **人物**：`[[周黎安]]`、`[[格申克龙]]`
- **案例**：`[[京东方案例]]`、`[[光伏产业案例]]`
- **其他章节**：`[[01-03 地方政府的权力与事务 - 复杂信息]]`

**标注频率**：每个概念在同一个章节文件中，**首次出现时标注，后续出现不重复标注**。避免满屏双括号影响阅读。

### wikilinks 统一词表

在开始拆分之前，必须先建立一份词表。这份词表贯穿整个项目，所有 Agent 都使用同一份。

词表建立步骤：
1. 读目录页，提取所有章节标题中的关键词
2. 读前言，提取作者提到的核心概念
3. 快速浏览每章开头，补充遗漏的概念
4. 统一命名：同一概念只用一个名字（不要"分税制"和"分税制改革"并存，选一个）

词表格式：
```
# 概念词表
分税制, 土地财政, 城投公司, 地方政府债务, 招商引资,
激励相容, 外部性, 规模经济, 信息不对称, 条块分割,
GDP锦标赛, 官员晋升锦标赛, 纵向不平衡, 横向不平衡,
产业引导基金, 土地金融, 户籍制度, 要素市场改革,
产能过剩, 国内大循环, 中美贸易冲突, 债务风险,
城市化, 房价, 贫富差距, 再分配

# 案例词表
京东方案例, 光伏产业案例, 合肥模式

# 人物词表
周黎安, 格申克龙, 张五常, 钱颖一
```

## 质量检查

章节拆分完成后，做以下检查：

1. **完整性** —— `ls chapters/ | wc -l` 文件数和目录中的节数一致
2. **连贯性** —— 随机打开 2-3 个文件，确认没有在句子中间截断
3. **wikilinks** —— 抽查几个文件，确认关键概念都被标注了
4. **命名一致性** —— 检查所有 wikilinks 引用的概念名是否在词表中
5. **frontmatter** —— 每个文件都有正确的 YAML frontmatter
