# lhx-obsidian-journal

帮助用户维护 Obsidian 每周日记文件。每周一个文件，早上写 Todo，晚上写复盘，周末做周复盘。

## 基本信息

- **Vault 路径**：`~/Documents/Obsidian Vault/`（自动适配当前用户，无需修改）
- **周文件目录**：`~/Documents/Obsidian Vault/{当前年份}/weekly/`
- **文件命名**：`{年份}-W{ISO周数}（{周一月.日}-{周日月.日}）.md`
  - 示例：`2026-W17（4.21-4.27）.md`
  - 月份和日期不补零，直接写数字

---

## 每次操作前：获取当前日期和路径

所有操作开始前，先运行此脚本：

```python
import datetime
from pathlib import Path

today = datetime.date.today()
year = today.year
week = today.isocalendar()[1]
weekday = today.weekday()  # 0=周一, 6=周日
weekday_names = ['周一','周二','周三','周四','周五','周六','周日']
monday = today - datetime.timedelta(days=weekday)
sunday = monday + datetime.timedelta(days=6)

vault = Path.home() / "Documents" / "Obsidian Vault"
weekly_dir = vault / str(year) / "weekly"
weekly_dir.mkdir(parents=True, exist_ok=True)

filename = f"{year}-W{week}（{monday.month}.{monday.day}-{sunday.month}.{sunday.day}）.md"
filepath = weekly_dir / filename

print(f"year={year}")
print(f"week={week}")
print(f"today={today.month}/{today.day}")
print(f"weekday={weekday_names[weekday]}")
print(f"monday={monday.month}/{monday.day}")
print(f"sunday={sunday.month}/{sunday.day}")
print(f"filename={filename}")
print(f"filepath={filepath}")
print(f"exists={filepath.exists()}")
```

用脚本输出的 `filepath` 作为操作目标，不要手动拼接路径，不要硬编码用户名或年份。

## 操作一：今日 Todo（早上用）

**触发词**：今日 Todo、今天任务、P1 任务、写任务清单

**步骤：**

1. 运行上方日期脚本，得到 `filepath` 和 `exists`
2. **文件不存在**（`exists=False`）：用【周文件模板】创建完整新文件（7天结构全部生成，每天填入正确日期）
3. **文件已存在**（`exists=True`）：找到今天对应的日期块（`## {weekday} {月/日}`），在其中的 `### 🎯 今日 Todo` 区域写入任务

任务格式：
```
#### P1 今天必须推进
- [ ] **[P1 | 截止 {时间}]** {任务描述}

#### P2 今天最好完成
- [ ] **[P2]** {任务描述}

#### 收尾检查
- [ ] 回看今天是否有闭环结果
- [ ] 明确明日最重要的 3 件事
```

## 操作二：今日复盘（晚上用）

**触发词**：今日复盘、写复盘、今天总结、晚上复盘

**步骤：**

1. 运行日期脚本，找到本周文件，定位今天的 `### 📝 今日复盘` 区域
2. 将用户提供的内容整理成结构化格式写入：

```
#### 事业
- 今日完成：{内容}
- 关键进展：{内容}
- 问题与阻塞：{内容}
- 明日重点：{内容}

#### 个人
- 运动 / 作息 / 饮食：{内容}
- 成长 / 阅读：{内容}

#### 财务
- {内容，无则填"无"}

#### 新发现 / 灵感
- {内容，无则填"无"}
```

用户口述时，帮他提炼整理，不要照搬原话。缺失的字段写"待补充"。

## 操作三：周复盘 - 自动模式

**触发词**：帮我生成周复盘、自动总结这周、AI 帮我做周复盘

**步骤：**

1. 读取本周文件的完整内容
2. 扫描每一天的 Todo 完成情况（`[x]` 表示完成）和复盘各字段
3. 归纳生成周总结，写入文件底部 `## 周复盘` 区块：

```
## 周复盘

### 本周完成情况
{从各天"今日完成"和"关键进展"提炼，3-5条最重要的事}

### 最大收获
{从各天"新发现/灵感"和"成长"提炼，2-3条}

### 下周最重要的一件事
{综合本周"问题与阻塞"和"明日重点"，给出一个明确的下周首要行动}
```

若 `## 周复盘` 区块已有内容，直接覆盖替换。

## 操作四：周复盘 - 手动模式

**触发词**：我来说周复盘、手动写周复盘、我自己总结这周

**步骤：**

1. 接收用户的口述内容
2. 整理成与自动模式相同的结构格式
3. 写入文件底部 `## 周复盘` 区块

## 周文件模板

创建新文件时，用日期脚本计算每天的具体日期，生成完整 7 天结构。

**文件标题格式**：`# {年份}-W{周数}（{周一月/日}-{周日月/日}）`
示例：`# 2026-W17（4/21-4/27）`

每天的块标题格式：`## 周一 4/21`、`## 周二 4/22`……

每天结构如下（7天全部生成）：

```
## {weekday} {月/日}

### 🎯 今日 Todo
#### P1 今天必须推进
- [ ] 

#### P2 今天最好完成
- [ ] 

#### 收尾检查
- [ ] 回看今天是否有闭环结果
- [ ] 明确明日最重要的 3 件事

### 📝 今日复盘
#### 事业
- 今日完成：
- 关键进展：
- 问题与阻塞：
- 明日重点：

#### 个人
- 运动 / 作息 / 饮食：
- 成长 / 阅读：

#### 财务
- 

#### 新发现 / 灵感
- 

---
```

文件最后加上：

```
## 周复盘

### 本周完成情况

### 最大收获

### 下周最重要的一件事
```

## 注意事项

- **路径始终动态获取**：每次都运行 Python 脚本，用 `Path.home()` 自动获取用户目录，永远不要硬编码路径或用户名
- **年份自动切换**：跨年时文件会自动存到新年份目录（如 `2027/weekly/`），无需手动调整
- **不重复创建文件**：文件已存在时只更新对应部分
- **精准定位**：用 `## 周X {日期}` 作为锚点定位今天的块
- **口述整理**：用户口述时主动提炼成结构化语言，不要原文照搬
- **周复盘覆盖**：`## 周复盘` 区块有旧内容时直接替换，不追加
