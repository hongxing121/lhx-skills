# 文章转短视频文案

将任意文章链接转换为可直接使用的短视频口播脚本。

## 语言

**跟随用户语言**：用户用中文则中文输出，用英文则英文输出。

## 参数说明

用户调用时可附带以下参数（均可选，未指定则询问或使用默认值）：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `url` | 必填 | 文章链接 |
| `platform` | 抖音 | 目标平台：抖音 / 视频号 / 小红书 / B站 |
| `duration` | 60s | 视频时长：60s / 90s / 3min / 5min |
| `style` | 口播 | 脚本风格：口播 / 图文解说 / 对话体 |

## 工作流程

### Step 1：确认参数

如果用户只提供了 URL，没有指定 platform / duration / style，**一次性**用 `AskUserQuestion` 询问（最多 3 个问题合并）。如果用户语气偏向"快速生成"或"帮我搞一下"，则**跳过询问**，使用默认值直接进行。

### Step 2：获取文章内容

根据 URL 类型选择抓取方式：

#### 2a. X / Twitter 链接（`x.com` 或 `twitter.com`）

检查 `baoyu-danger-x-to-markdown` skill 是否存在：

```bash
test -f ~/.claude/skills/baoyu-danger-x-to-markdown/SKILL.md && echo "found" || echo "not found"
```

- **找到**：调用该 skill 转换内容
- **未找到**：用 WebFetch 直接抓取页面文本

#### 2b. 微信公众号链接（`mp.weixin.qq.com`）

检查 `baoyu-url-to-markdown` skill 是否存在：

```bash
test -f ~/.claude/skills/baoyu-url-to-markdown/SKILL.md && echo "found" || echo "not found"
```

- **找到**：调用该 skill，使用 `--wait` 模式（微信文章需要渲染）
- **未找到**：用 WebFetch 抓取，提醒用户部分内容可能不完整

#### 2c. 其他任意 URL

检查 `baoyu-url-to-markdown` skill 是否存在：

```bash
test -f ~/.claude/skills/baoyu-url-to-markdown/SKILL.md && echo "found" || echo "not found"
```

- **找到**：调用该 skill 转换内容
- **未找到**：用 WebFetch 直接抓取

#### 抓取失败处理

如果抓取失败或内容过少（< 200 字），告知用户并请求直接粘贴文章正文，然后继续 Step 3。

### Step 3：理解文章

在生成脚本前，先内部完成以下分析（**不输出给用户**）：

1. **核心主题**：文章在讲什么？一句话概括
2. **关键信息点**：3-5 个最有价值的信息，按重要性排序
3. **目标受众**：文章面向谁？他们关心什么？
4. **情绪基调**：信息类 / 观点类 / 故事类 / 干货类
5. **可视化潜力**：哪些内容适合配画面或字幕强调

### Step 4：生成脚本

根据平台和时长的对应关系控制字数：

| 平台 | 60s | 90s | 3min | 5min |
|------|-----|-----|------|------|
| 抖音/视频号 | 150字 | 220字 | 450字 | 750字 |
| 小红书 | 120字 | 180字 | — | — |
| B站 | 180字 | 270字 | 540字 | 900字 |

**输出格式如下**：

---

## 📱 [平台] 短视频脚本

**视频标题**：[5 个备选标题，带数字/悬念/利益点，适合该平台风格]

**封面文案**：[一句话，适合作为封面或首帧字幕，≤15字]

---

### 🎬 脚本正文

> 格式说明：每段标注 `[时间范围]` 和 `[时长]`，口播内容直接可读，制作备注用 `*斜体*` 标注

---

**[0s–5s] 开场钩子** ⏱ 5s

[直接开口的第一句话，必须在 3 秒内制造悬念、共鸣或利益感，禁止"大家好我是xxx"式开场]

*制作备注：建议配[画面/字幕/音效]描述*

---

**[5s–Xs] [段落名称]** ⏱ Xs

[口播正文，自然口语化，适合朗读，避免书面语和长句]

*制作备注：[可选的画面/字幕建议]*

---

（按时长继续拆分段落，每段不超过 60 字）

---

**[最后 5s–10s] 结尾引导** ⏱ 5–10s

[引导关注/点赞/评论/收藏的话术，自然不强硬]

---

### 🏷️ 话题标签

[针对该平台的 5-8 个话题标签]

### 📝 制作建议

- **封面**：[封面设计建议]
- **BGM**：[音乐风格建议]
- **剪辑节奏**：[快切 / 慢节奏 / 混合]
- **注意**：[文章中哪些内容不适合直接引用，需要改写或跳过]

---

### Step 5：询问调整

脚本输出完毕后，询问：

> 需要调整吗？可以告诉我：换个平台、改变时长、调整风格（更幽默/更严肃/更口语），或者对某一段重写。

## 各平台脚本风格指南

### 抖音
- 前 3 秒必须有钩子，否则用户划走
- 语速快，节奏感强，多用短句
- 强悍的结论优先，逻辑在后
- 口语化：「你知道吗」「真的吗」「没想到吧」
- 禁用："首先、其次、最后"等书面列举

### 视频号
- 相对抖音更沉稳，适合中年用户
- 故事性开场效果好
- 适当保留思考空间，不需要太快
- 结尾引导分享比点赞更有效

### 小红书
- 标题带 emoji，有代入感
- 开头"姐妹们！""打工人必看！"等强共鸣词
- 干货点用分点列举
- 结尾引导评论互动

### B站
- 允许较长铺垫，用户耐心更好
- 可以有更完整的逻辑链
- 幽默和梗文化接受度高
- 结尾"三连"引导

## 注意事项

- **不编造**：脚本内容必须忠实于原文，不能添加原文中没有的数据或观点
- **不照搬**：将书面语自然转化为口语，不能直接复制原文段落
- **版权意识**：如原文有明确版权声明，在制作建议中提醒用户注意引用规范
- **敏感内容**：遇到政治敏感、争议性内容，在制作建议中标注风险
